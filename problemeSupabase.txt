
Oui, tu peux cliquer sur "Set up custom SMTP server" dans Supabase pour configurer ton propre serveur d’envoi d’emails.
Cela te permettra de lever les limites d’envoi et d’assurer la délivrabilité des emails (inscription, reset password, etc.).

Voici comment faire :

Clique sur "Set up custom SMTP server" dans Supabase (Auth > Settings > Email).

Renseigne les informations de ton fournisseur SMTP (Gmail, Outlook, Mailgun, Sendgrid, etc.) :

SMTP sender email : l’adresse qui enverra les mails (ex : tonadresse@gmail.com)
SMTP server : l’adresse du serveur SMTP (ex : smtp.gmail.com)
SMTP port : souvent 465 (SSL) ou 587 (TLS)
SMTP username : ton adresse email ou identifiant
SMTP password : le mot de passe de l’application (pour Gmail, il faut générer un mot de passe d’application)
Secure : coche si tu utilises SSL/TLS
Sauvegarde la configuration.

Attention :

Si tu utilises Gmail, il faut activer la validation en deux étapes et générer un mot de passe d’application (pas ton mot de passe habituel).
Pour d’autres services (Mailgun, Sendgrid…), utilise les identifiants fournis par le service.
Une fois configuré, refais un test d’inscription.
Si tu veux, donne-moi le fournisseur que tu veux utiliser (Gmail, autre…) et je te guide pas à pas pour remplir chaque champ !
